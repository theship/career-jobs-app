## Instructions:
1. ONLY extract skills that are explicitly mentioned or strongly implied in the resume text
2. You MUST use ONLY skills from the predefined vocabulary - do not create new skills
3. Provide confidence scores (0.0 to 1.0) based on:
   - 1.0: Skill is explicitly named in the text
   - 0.8-0.9: Skill is strongly implied by context (e.g., "built REST APIs" implies REST)
   - 0.6-0.7: Skill is somewhat implied but not directly stated
   - Below 0.6: Do not include (too uncertain)

4. Extract years of experience if mentioned near the skill, for example:
   - "5 years of Python experience" → Python: 5 years
   - "Senior Java developer (3+ years)" → Java: 3 years
   - Look within 20 words of the skill mention for experience indicators

5. DO NOT hallucinate or add skills that are not clearly present in the text
6. DO NOT add skills just because they are commonly paired with mentioned skills
7. Focus on technical skills, tools, frameworks, and methodologies

## Examples of valid extractions:
- Text: "Developed RESTful APIs using Python and FastAPI" 
  → Extract: Python (1.0), FastAPI (1.0), REST (0.9)
  
- Text: "Led agile team of 5 engineers"
  → Extract: Agile (1.0), Leadership (0.8)

- Text: "Built machine learning models for customer churn prediction"
  → Extract: Machine Learning (1.0)

## Examples of invalid extractions:
- Text: "Worked on web application"
  → DO NOT extract: JavaScript, HTML, CSS (not mentioned)
  
- Text: "Used Python"
  → DO NOT extract: Django, Flask, NumPy (not mentioned, even if common with Python)
